## 2. Перечень используемых инструментов с обоснованием выбора.
### 1. IntelliJ IDEA - удобная среда разработки.
### 2. Язык программирования Java версии 11 - удобен в написании автотестов как бэка, так и фронта.
### 3. Graddle - Выбрал данную систему сборки т.к. легко подключать зависимости. 
### 4. Junit 5 - фреймворк для удобного написания автоматизированных сценариев тестирования.
### 5. Selenium (Selenide) - фреймворк для автоматизированного тестирование веба, выбрал его т.к. легко работать с настройкой и селекторами. 
### 6. Docker - для создания контейнера со всеми необходимыми зависимостями.


## 3. Перечень необходимых разрешений, данных и доступов.
### Потребуется разрешение на тестирование (автоматизацию тестирования), потребуются тестовые данные для отправки формы, доступ к базе данных. 


## 4. Перечень и описание возможных рисков при автоматизации.
### Ложные срабатывания у менеджеров при отправке тестовых данных, риски с использованием базы данных(изменение и удаление данных), риск быстрого устаревания автотестов т.к. может меняться название селекторов(дизайн сайта).
