# План автоматизации тестирования формы записи на обучение по профессии "Инженер по тестированию"


## 1. Перечень автоматизируемых сценариев.

### Валидные данные для заполнения формы:
### Имя - Павел Павлов
### Телефон - +7 999 000 00 00

### Невалидные данные для заполнения формы:
### Имя - +++++++++ , пустое поле
### Телефон - +7 999 , пустое поле 

### 1.1 Отправка заявки на обучение с валидными данными. 
#### Предусловие: На странице [Нетологии](https://netology.ru/), в верхней части страницы нажать на "Каталог курсов" и в поиске ввести "Инженер по тестированию", нажать на появившийся курс под строкой поиска.
![1](https://github.com/Smytstone/-/assets/147393251/27dabe43-1337-47a4-8f96-46bad43b4679)
#### На [странице](https://netology.ru/programs/qa#/form_guarantees) пролистать до формы заявки на обучение. Заполнить поля "Имя", "Телефон" валидными данными и нажать кнопку "Записаться".
![1234](https://github.com/Smytstone/-/assets/147393251/5b10c491-06a5-471d-88c4-caadba0345f4)
#### Ожидаемый результат: Появляется окно с сообщением "Вы записались на курс Инженер по тестированию". 

### 1.2 Отправка заявки на обучение с валидным именем и невалидным телефоном. 
#### Предусловие: На странице [Нетологии](https://netology.ru/), в верхней части страницы нажать на "Каталог курсов" и в поиске ввести "Инженер", нажать на кнопку "Найти курс" справа от строки поиска. На открывшейся [странице](https://netology.ru/navigation?query=%D0%98%D0%BD%D0%B6%D0%B5%D0%BD%D0%B5%D1%80) выбрать нужный курс.
![2](https://github.com/Smytstone/-/assets/147393251/c63cb3e2-89e5-49f7-9706-b9a764877228)
#### На [странице](https://netology.ru/programs/qa#/form_guarantees) пролистать до формы заявки на обучение. Заполнить поле "Имя" валидными данными, в поле "Телефон" ввести невалидные данные и нажать кнопку "Записаться".
![1234](https://github.com/Smytstone/-/assets/147393251/5b10c491-06a5-471d-88c4-caadba0345f4)
#### Ожидаемый результат: Поле "Телефон" обводится красной рамкой и под полем появляется надпись "Неверный формат".

## 2. Перечень используемых инструментов с обоснованием выбора.
### 1. IntelliJ IDEA - удобная среда разработки.
### 2. Язык программирования Java версии 11 - удобен в написании автотестов как бэка, так и фронта.
### 3. Graddle - Выбрал данную систему сборки т.к. легко подключать зависимости. 
### 4. Junit 5 - фреймворк для удобного написания автоматизированных сценариев тестирования.
### 5. Selenium (Selenide) - фреймворк для автоматизированного тестирование веба, выбрал его т.к. легко работать с настройкой и селекторами. 
### 6. Docker - для создания контейнера со всеми необходимыми зависимостями.


## 3. Перечень необходимых разрешений, данных и доступов.
### Потребуется разрешение на тестирование (автоматизацию тестирования), потребуются тестовые данные для отправки формы, доступ к базе данных.
